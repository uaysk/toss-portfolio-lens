남아 있는 경계

측정 결과를 과장하지 않기 위해 현재 구조의 제약과 다음 최적화 지점을 함께 기록합니다.

01 / CANCEL협력적 checkpoint 취소

명령 단위 선점은 아니지만 ledger 일자·후보 batch·scenario/fold·Monte Carlo step의 안전 경계에서 중단합니다. 실제 PostgreSQL running-job 취소는 308ms, output artifact 0건으로 검증했습니다.

02 / SERIALIZERaw JSON 비용

큰 백테스트 결과는 Rust 약 705KB입니다. UDS 왕복의 남은 비용은 JSON encode/decode와 메모리 복사이며 binary/columnar 전송은 후속 후보입니다.

03 / ROLLBACKNode legacy 경로 존치

즉시 롤백을 위한 기존 inline 엔진은 명시적 legacy 모드로 남습니다. 기본 production 계산 경로로 간주하지 않습니다.

04 / RELEASE운영 배포 전 상태

구현과 로컬 검증은 완료했지만 commit·push·merge·운영 배포는 수행하지 않았습니다. 실제 운영 부하와 observability 검증이 남습니다.

Final decision

정확한 ledger와 반복 탐색 성능을 Rust에 모으고, Node는 얇고 관찰 가능한 control plane으로 유지합니다.
