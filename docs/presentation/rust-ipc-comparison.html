Rust low-latency path proposal

계산보다 긴 대기열을
요청 경로에서 걷어냅니다.

빠른 단일 백테스트는 Unix socket으로 Rust에 직접 전달하고, 내구성이 필요한 최적화는 기존 PostgreSQL 큐에 남깁니다. 언어 교체보다 전송 경로 분리가 실제 체감 속도를 결정합니다.

읽는 방법

현재 수치는 실제 로컬 벤치마크입니다. Rust·IPC 수치는 같은 하드웨어를 전제로 한 설계 추정치이며, 아래 슬라이더로 보수적·기준·공격적 가정을 직접 바꿀 수 있습니다.

Isolation
프로세스 격리 유지

N-API가 아닌 Rust sidecar를 쓰면 계산 실패가 Node API 프로세스를 직접 종료하지 않습니다.

Durability
작업별 이중 경로

빠른 백테스트는 IPC, 재시도·감사가 중요한 대규모 최적화는 PostgreSQL 큐를 유지합니다.

Complexity
계약 하나, 전송 둘

동일한 버전 계약과 checksum을 공유하되 socket backpressure·timeout·재연결 정책이 추가됩니다.
